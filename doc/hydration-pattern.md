# Removed pattern: site-builder hydration

This document records the removed hydration pattern so it can be recovered from git if needed.

## What existed

The old pattern used:

- `hydration.tsx` next to a template
- generated `hydration.compiled.js`
- a site-builder `HydrationManager`
- a template `interactive` field containing the compiled client script

## Compile step

Hydration scripts were generated by:

- `scripts/compile-hydration.ts`
- wired through the repo `precompile` task
- exposed via `brain-precompile`

The compiler scanned for `**/templates/*/hydration.tsx` and emitted sibling `hydration.compiled.js` files.

## Runtime contract

At runtime, site-builder collected the template `interactive` script and injected hydration assets into rendered HTML.

That contract depended on:

- `window.preact`
- route-level hydration data script injection
- post-processing HTML output after SSR

## Why it was removed

The repo no longer had meaningful consumers for this machinery.

The only real consumer was the old dashboard template path, while the dashboard already rendered correctly through its direct web route.

Keeping hydration around created:

- duplicate dashboard delivery paths
- dead build plumbing
- extra startup/build complexity

This removal was about one specific implementation, not a blanket rejection of future client-side interactivity.

## Recovery

If hydration is ever needed again, recover it from git history before this cleanup series.

Reference commit range:

- dashboard cleanup starts after commit `524c407f`
