---
title: Plugin System
---
The plugin system is the primary extension mechanism for the brain application, letting it grow without changing core logic. It defines how plugins are discovered, loaded, registered, and integrated, with stable boundaries between core and extension code. The architecture includes distinct plugin families such as EntityPlugin, ServicePlugin, InterfacePlugin, and MessageInterfacePlugin: entity plugins define durable markdown-backed content types, service plugins add tools and integrations, and interface plugins expose transports and routes such as MCP, Discord, A2A, and webserver endpoints. Relevant documentation covers lifecycle hooks, authoring and registration patterns, external plugin loading through `brain.yaml`, and the use of public `@rizom/brain/*` exports for third-party packages. Overall, the plugin system is a central architectural feature for extensibility, modularity, and long-term evolution.
