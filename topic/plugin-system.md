---
title: Plugin System
---
A plugin system is a modular extension architecture that adds new behavior to an application without modifying its core code. It defines stable extension points for discovering, loading, registering, and integrating plugins, with clear contracts that preserve separation between core logic and add-on functionality. This supports extensibility, maintainability, compatibility, and safer evolution as the platform grows.

In this brain system, the plugin system is the primary mechanism for introducing specialized capabilities while keeping the base architecture lightweight. It governs what stays in the core versus what is delegated to plugins, and it provides a structured way to scale functionality over time without increasing complexity in the main codebase. Common plugin families include entity plugins for durable markdown-backed content types, service plugins for tools and integrations, and interface plugins for transports and routes such as MCP, Discord, A2A, and webserver endpoints. Related concerns include lifecycle hooks, authoring and registration patterns, external loading through `brain.yaml`, and public `@rizom/brain/*` exports for third-party packages.
